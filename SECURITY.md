# Security Policy

## Supported Versions

Security fixes are handled on the default branch until a public release policy is established.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability.

Report security issues privately to the project maintainers. Include:

- Affected version or commit.
- Steps to reproduce.
- Impact and affected components.
- Any relevant logs with secrets removed.

Maintainers will acknowledge the report and coordinate a fix before public disclosure.
